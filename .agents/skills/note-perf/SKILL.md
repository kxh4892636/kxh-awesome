# Note Perf

优化已有技术笔记，使其符合 write-note 的格式与语言规范。核心约束：**优化后的笔记必须包含原笔记的所有知识点**。

## 核心约束

1. **知识点零丢失**：原笔记中每个独立知识点在优化后必须存在，不可删除任何知识内容
2. **只改形式不改内容**：优化的是表达方式、格式结构、语言风格，不是知识本身
3. **可删冗余不可删信息**：可以删除废话、重复表述、过渡语句，但不能删除任何实质知识点

## 优化维度

### 1. 语言精简

将完整句子压缩为电报体短语，每条 bullet 以分号 `;` 结尾。

| 优化前                                                                                       | 优化后                                                               |
| -------------------------------------------------------------------------------------------- | -------------------------------------------------------------------- |
| Date.parse() 是一个静态方法，它接收一个日期字符串参数，然后返回对应的毫秒级时间戳            | Date.parse(dateString) 静态方法: 返回 dateString 对应的毫秒级时间戳; |
| 在浏览器环境中，window 对象实际上是 global object 的一个代理对象                             | 浏览器中 global object 的代理;                                       |
| 由于数组存在负数，正数与负数相乘，会让最大乘积变最小乘积，反之依然，故维护数组最大值和最小值 | 数组存在负数, 正负相乘会翻转大小, 故维护 max 和 min 两个 dp;         |
| 如果鸡蛋没碎，i=i，j=j-1，最多可以检查 dp[i][j-1] 层楼层                                     | 如果鸡蛋没碎, i=i, j=j-1, 最多检查 dp[i][j-1] 层;                    |

### 2. 格式规范化

- bullet 结尾统一为分号 `;`（非句号、非无标点）
- 关键词/术语前置，描述紧跟其后
- API 签名统一为 TypeScript 风格：`Type.method(param: Type): ReturnType: 描述;`
- 代码块添加语言标记
- 图片使用相对路径：`![alt](images/filename.png)`
- 数学公式使用 LaTeX：行间 `$$...$$`，行内 `$...$`

### 3. 结构优化

- 标题层级规范：H1(标题) → H2(大主题) → H3(子主题) → H4(子主题/具体概念) → H5(具体概念/API 分组)，最多到 H5
- **层级必须连续递进，禁止跳级**（如 H3 直接跳 H5 应改为 H3 → H4）
- 删除引言/总结/过渡段落
- 删除"下面介绍..."等引导语
- 删除重复标题已表达的信息
- 合并同一知识点的多种表述为一种
- 将散落的同类信息归入同一 H5 分组（如：将散落的方法签名归入"实例方法"分组）

### 4. 表格化

以下场景将 bullet 转为表格：

- 属性/方法枚举（概念性）
- 对比（如 TCP vs UDP）
- 协议/规范要求
- 运算符/表达式枚举（独立于代码示例时）

**禁止表格化的场景**：

- 与代码强相关的并列知识点（如关键字含义、修饰符语义、类型系统枚举、工具类型清单）
- 判断准则：若每条知识点对应一行可演示的代码，则首选"代码块 + 注释"形式
- 此类知识点应转入第 5 维度（代码注释化）

### 5. 代码注释化（核心维度）

将与代码强相关的并列 bullet 知识点合并入代码块的注释，按用途选择注释位置：

| 注释类型     | 用途                           | 位置                         |
| ------------ | ------------------------------ | ---------------------------- |
| **代码解释** | 行为说明、运行结果、错误提示   | 行内（代码末尾 `//`）        |
| **知识讲解** | 概念定义、语法说明、关键字含义 | 单独一行（在被解释代码上方） |

**优化前（散落 bullet + 代码块）**：

```markdown
- `boolean`: 布尔值
- `number`: 数值
- `string`: 字符串

\`\`\`typescript
const isDone: boolean = false;
const num: number = 1;
const myName: string = "Tom";
\`\`\`
```

**优化后（合并入代码注释）**：

````markdown
```typescript
// 布尔值
const isDone: boolean = false;
// 数值
const num: number = 1;
// 字符串
const myName: string = "Tom";
```
````

**优化前（行为说明用单独 bullet）**：

```markdown
- handleEvent 报错, event 不能为 'dblclick';

\`\`\`typescript
handleEvent(document.getElementById("world"), "dblclick");
\`\`\`
```

**优化后（行为说明用行内注释）**：

````markdown
```typescript
handleEvent(document.getElementById("world"), "dblclick"); // 报错, event 不能为 'dblclick'
```
````

### 6. 代码块精简

- 删除非必要的代码行
- 删除已由注释承载的冗余说明

## 优化流程

1. **提取知识点**：逐条提取原笔记中所有独立知识点，建立清单
2. **优化语言**：将完整句子压缩为电报体短语，统一分号结尾
3. **规范格式**：API 签名、代码块标记、图片路径、数学公式
4. **优化结构**：调整标题层级（连续递进，禁止跳级），删除过渡段落，合并同类信息
5. **表格化**：将适合的概念性 bullet 转为表格（排除与代码强相关的）
6. **代码注释化**：将与代码强相关的并列 bullet 合并入代码块注释
   - 知识讲解 → 单独一行注释（在被解释代码上方）
   - 代码解释 → 行内注释
7. **精简代码**：删除非必要的代码行
8. **完整性校验**：逐条对照步骤 1 的知识点清单，确认每个知识点在优化后均存在

## 完整性校验方法

优化完成后，执行以下校验：

1. 读取原笔记，提取所有独立知识点（每个 bullet、每个代码示例、每个表格行、每个图片引用均为一个知识点）
2. 逐条检查优化后笔记是否包含该知识点
3. 如有遗漏，将遗漏知识点补充回优化后笔记
4. 重复检查直到零遗漏

## 禁止事项

- 不删除任何知识点（即使看起来不重要）
- 不改变知识点的技术含义
- 不合并不同知识点为同一条 bullet
- 不添加原笔记中不存在的新知识点
- 不加 emoji
- 不重写代码逻辑
- 不将与代码强相关的并列知识点用表格或代码外的 bullet 列出（应合并入代码注释）
- 不混用注释位置（代码解释统一用行内，知识讲解统一用上方独立行）
- 不跳级使用标题（H3 不能直接跳 H5）
