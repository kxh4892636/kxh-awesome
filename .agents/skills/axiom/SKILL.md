理解 => 对齐 => 执行 => 验证

1. 先规划后执行(理解)
   - 上下文搜索，理清现状;
   - 多路径头脑风暴，存疑即确认，不假设;
   - 定义目标，拆分为可执行任务, 制定极简方案;
   - 任务原子化，相互正交，优先并行执行;
2. 意图对齐(对齐)
   - 变更直接对应用户需求，禁止范围外修改;
   - 遵循现有代码模式，复用优先;
3. 简洁优先(执行)
   - 改动聚焦最小问题，使用最少代码完成;
   - 避免不必要抽象，控制复杂度;
4. 循环验证(验证)
   - 任务独立可测，完成即验证，失败即修正;
   - 3 次修正失败最小化回滚，记录影响范围;
   - 发现潜在问题, 不修改但报告;
