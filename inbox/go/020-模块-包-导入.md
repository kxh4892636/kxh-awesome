# 模块-包-导入

## Module

### 基本概念

#### 定义

- module: 一组共享同一 module path 的 Go package 集合;
- `go.mod`: module 配置文件, 记录 module path、Go 版本、直接依赖;
- `go.sum`: 依赖内容校验文件, 用于校验下载内容未被篡改;
- module path: 当前 module 被导入时使用的根路径;

### 创建模块

#### 命令

```bash
mkdir hello # 创建项目目录
cd hello # 进入项目目录
go mod init example.com/hello # 创建 go.mod 并写入 module path
```

### 依赖管理

#### 命令

```bash
go get github.com/google/uuid@latest # 添加或升级依赖到最新版本
go get github.com/google/uuid@none # 删除指定依赖
go mod tidy # 根据源码 import 补齐和清理 go.mod/go.sum
```

### replace

#### 语法格式

```go
// 将 example.com/lib 替换为本地 ../lib; 常用于本地联调或临时 fork
replace example.com/lib => ../lib
```

### 版本规则

#### 概念

- `v0`: 初始阶段, 兼容性不稳定;
- `v1`: 默认稳定主版本, import path 不需要版本后缀;
- `v2+`: 主版本通常体现在 module path 中, 如 `/v2`;

#### Semantic Versioning

- X.Y.Z: 主版本号.次版本号.修订版本号;
- major version: 不兼容变更, 如新增大功能、修改 API;
- minor version: 兼容功能新增, 如新增小功能、修复小问题;
- patch version: 兼容问题修复, 如修复小问题、优化性能;

## Package

### 基本概念

#### 定义

- package: Go 代码的基本组织单位;
- 目录与包: 一个目录通常对应一个 package;
- 包名: 源文件 `package name` 声明的名字;
- 导入路径: import 使用的字符串路径, 不等于包名本身;

### main 包

#### 语法格式

```go
// main 包表示当前 package 可编译为可执行程序
package main

// main 函数为程序入口; main 返回后进程结束
func main() {
	println("start")
}
```

### 普通包

#### 语法格式

```go
// calc 为普通库包; 不能单独作为可执行程序入口
package calc

// Add 首字母大写; 可被其他 package 访问
func Add(a int, b int) int {
	return a + b
}
```

### init 函数

#### 语法格式

```go
package config

var Loaded bool

// init 在包初始化时自动调用一次; 不能被代码显式调用
func init() {
	Loaded = true
}
```

#### 执行顺序

- 被导入包: 先于当前包初始化;
- 当前包变量: 先于当前包 `init` 初始化;
- 多个 `init`: 按文件和声明顺序执行;

## Import

### 基本导入

#### 语法格式

```go
package main

// 导入标准库 fmt 包
import "fmt"

func main() {
	fmt.Println("hello") // 通过包名访问导出标识符
}
```

### 分组导入

#### 概念

- 分组导入: 使用括号组织多个 import;
- gofmt: 自动排序和整理 import;

#### 语法格式

```go
import (
	"fmt"       // 标准库包
	"math/rand" // 标准库包
)
```

### 别名导入

#### 语法格式

```go
// random 为当前文件内使用的导入别名
import random "math/rand"

func main() {
	println(random.Intn(10))
}
```

### 空导入

#### 语法格式

```go
// 空导入只触发目标包 init 副作用, 不直接使用导出标识符
import _ "database/sql"
```

### 点导入

#### 语法格式

```go
// 点导入将 fmt 的导出标识符放入当前文件作用域; 普通业务代码不推荐
import . "fmt"

func main() {
	Println("hello")
}
```

## 可见性

### 导出规则

#### 概念

- 导出标识符: 首字母大写, 包外可见;
- 非导出标识符: 首字母小写, 仅包内可见;
- 可见性粒度: package, 不是文件;

#### 语法格式

```go
package user

type Profile struct {
	Name string // 首字母大写字段; 包外可见
	age  int    // 首字母小写字段; 仅包内可见
}
```

### internal 包

#### 概念

- `internal`: 受 Go 工具链限制的内部包目录;
- 导入限制: 只有 `internal` 父目录及其子目录下的包可导入;
- `cmd`: 约定放置可执行程序入口;
- `pkg`: 约定放置可被外部复用的公共包;

#### 目录结构

```text
project/
├── go.mod
├── cmd/
│   └── app/
│       └── main.go
├── internal/
│   └── service/
│       └── user.go
└── pkg/
    └── id/
        └── id.go
```

## 项目结构

### 小型项目

#### 概念

- 小型项目: 优先保持结构简单;
- 拆包时机: 职责稳定后再拆分 package;

#### 目录结构

```text
hello/
├── go.mod      # module 配置文件
├── go.sum      # 依赖校验文件
├── main.go     # 可执行程序入口
└── calc/       # 普通业务包目录
    └── calc.go # calc 包源码
```

### 常见分层

#### 目录结构

```text
project/
├── cmd/               # 可执行程序入口集合
│   └── server/        # server 程序入口目录
│       └── main.go    # server main 包入口文件
├── internal/          # 仅当前 module 内部可导入的代码
│   ├── handler/       # 输入输出边界层
│   ├── service/       # 业务规则层
│   └── repository/    # 数据访问层
├── pkg/               # 可被外部 module 复用的公共包
└── go.mod             # module 配置文件
```
