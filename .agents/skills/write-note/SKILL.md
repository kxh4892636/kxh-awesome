# Write Note

创建第一性原理驱动的技术笔记。只保留本质概念与关键信息，删除一切冗余。

## 核心原则

1. **第一性原理**：只记录不可再拆解的核心概念和关键行为，不记录可推导的结论
2. **信息密度最大化**：每个 bullet 承载一个独立知识点，不写废话
3. **关键词前置**：重要术语/概念放在 bullet 开头，描述跟在后面
4. **中英混排**：技术术语保留英文原文，描述用中文

## 文件结构

```markdown
# 标题

## 大章节 (H2)

### 子章节 (H3)

##### 子章节 (H4)

##### 具体概念 (H5)

- 知识点;
  - 补充细节;
```

### 标题层级规则

| 层级 | 用途 | 示例 |
|------|------|------|
| H1 | 文档标题，仅一个 | `# 基本引用类型` |
| H2 | 大主题分区 | `## The Global Object` |
| H3/H4/H5 | 子主题/具体概念 | `##### 实例属性` |

最多使用 H5 层级

## 内容格式规则

### Bullet Points

- 每条以分号 `;` 结尾（非句号）
- 不写完整句子，用电报体短语
- 缩进 sub-bullet 补充细节或例外情况
- 一条 bullet = 一个知识点，不合并

```markdown
- 无法显式访问;
- 所有可以被全局访问的变量和函数为其属性;
```

### 表格

用于对比、枚举、属性列表等结构化数据：

```markdown
| 属性 | 描述 |
|------|------|
| Math.PI | PI |
| Math.E | e |
```

### 代码块

- 添加语言标记，如 `typescript`、`go`、`python`、`java` 等
- 只展示最小必要示例，不写解释性注释（除非注释本身是知识点）
- 注释用 `//` 行内标注关键行为

```typescript
// g 匹配全部内容
let pattern1 = /at/g;
```

### API 签名

使用 TypeScript 风格的类型标注，格式：

```markdown
- ClassName.method(param: Type): ReturnType: 描述;
- ClassName.property: Type 属性: 描述;
```

示例：

```markdown
- Number.isInteger(number: number): boolean 静态方法: 判断 number 是否为 int 类型;
- String.length: number 属性: 返回字符串 code unit 数量;
```

## 禁止事项

- 不写引言/总结/过渡段落
- 不写"下面介绍..."等引导语
- 不重复标题已表达的信息
- 不写显而易见的内容
- 不加 emoji
- 不写完整句子（用短语片段）
- 不对同一知识点做多种表述

## 写作流程

1. 识别源材料的核心概念和关键 API
2. 按逻辑关系组织标题层级（H2 → H3 → H5）
3. 每个概念提炼为最短 bullet，附带类型签名
4. 对比/枚举类信息用表格
5. 需要展示用法的用最小代码块
6. 检查：删除所有可以不写的字词，直到无法再删

## 语言风格对照

| 不要 | 要 |
|------|------|
| Date.parse() 是一个静态方法，它接收一个日期字符串参数，然后返回对应的毫秒级时间戳 | Date.parse(dateString) 静态方法: 返回 dateString 对应的毫秒级时间戳; |
| 在浏览器环境中，window 对象实际上是 global object 的一个代理对象 | 浏览器中 global object 的代理; |
| 下面我们来看一下如何获取 Global object | ##### 获取 Global object |
