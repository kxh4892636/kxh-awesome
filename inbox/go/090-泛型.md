# 泛型

## 类型参数

### 基本概念

#### 定义

- generic: 声明时保留类型占位符, 使用时再指定具体类型;
- type parameter: 类型参数, 写在方括号中;
- type argument: 调用或实例化时提供的具体类型;
- constraint: 类型参数必须满足的约束;

### 泛型函数

#### 语法格式

```go
func First[T any](items []T) T { // T 为类型参数; any 为约束
	return items[0] // 返回值类型为 T
}
```

### 调用泛型函数

#### 概念

- 类型推导: 编译器可根据普通参数推导类型参数;
- 推导失败: 需要显式写出类型实参;

#### 语法格式

```go
func main() {
	a := First[int]([]int{1, 2, 3})    // 显式提供类型实参 int
	b := First([]string{"go", "lang"}) // 编译器根据参数推导 T 为 string
	println(a, b)
}
```

## 约束

### any

#### 概念

- `any`: `interface{}` 的别名;
- 操作限制: 只可使用所有类型都支持的操作;

#### 语法格式

```go
func Identity[T any](v T) T { // any 允许任意类型作为类型实参
	return v
}
```

### comparable

#### 概念

- `comparable`: 允许相等比较的类型约束;
- 不包含类型: slice、map、function;

#### 语法格式

```go
func Equal[T comparable](a, b T) bool { // comparable 允许 == 和 !=
	return a == b
}
```

### 自定义约束

#### 概念

- constraint interface: 可在接口中写类型集合;
- union: `|` 表示允许多个类型;
- 类型集合: 约束允许的所有类型;

#### 语法格式

```go
type Integer interface {
	~int | ~int8 | ~int16 | ~int32 | ~int64 // ~T 允许底层类型为 T 的自定义类型
}

func Max[T Integer](a, b T) T {
	if a > b { // 只有类型集合中所有类型都支持的操作可用
		return a
	}
	return b
}
```

### 类型元素

#### 概念

- 基本接口: 只包含方法的接口, 可作为普通值类型;
- 非基本接口: 包含类型元素的接口, 只能作为约束使用;

#### 语法格式

```go
type Number interface {
	~int | ~int64 | ~float64 // 类型元素只能用于约束接口
}
```

## 泛型类型

### 定义

#### 语法格式

```go
type Stack[T any] struct {
	items []T // 字段类型可使用类型参数
}
```

### 方法

#### 概念

- receiver 类型参数: 方法 receiver 需要带上类型参数;
- 方法级类型参数: Go 方法不能声明自己的额外类型参数;

#### 语法格式

```go
func (s *Stack[T]) Push(v T) {
	s.items = append(s.items, v)
}

func (s *Stack[T]) Pop() (T, bool) {
	var zero T // 获取类型参数 T 的零值
	if len(s.items) == 0 {
		return zero, false
	}
	last := len(s.items) - 1
	v := s.items[last]
	s.items = s.items[:last]
	return v, true
}
```

### 使用

#### 语法格式

```go
func main() {
	var s Stack[int] // 用 int 实例化泛型类型
	s.Push(10)
	v, ok := s.Pop()
	println(v, ok)
}
```

### 递归引用

#### 概念

- 泛型类型递归: 类型声明内部可引用自身的实例化形式;
- 类型实参: 递归引用时需要提供类型参数;
- 使用场景: 链表、树、图等递归数据结构;

#### 语法格式

```go
type Node[T any] struct {
	Value T
	Next  *Node[T] // 递归引用同一类型实参的 Node
}
```

## 泛型设计

### 使用场景

#### 概念

- 容器结构: stack、set、queue 等持有任意元素类型的数据结构;
- 算法复用: 对多种数值类型执行相同逻辑;
- 类型安全: 避免使用 `any` 后再做类型断言;

### 不适用场景

#### 概念

- 单一类型逻辑: 直接写普通函数更清晰;
- 约束复杂: 需要大量类型分支时泛型收益较低;
- 业务抽象: 优先使用具体类型和接口表达行为;

### 接口与泛型

#### 对比

| 机制 | 关注点 |
| ---- | ------ |
| interface | 值能做什么行为 |
| generic | 代码能适配哪些类型 |
