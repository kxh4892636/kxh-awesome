# 云服务器

## SSH 配置

### 基本概念

SSH 登录认证有两种方式：

| 方式 | 安全 | 描述 |
|------|------|------|
| 密码登录 | 低 | 输入密码，易被暴力破解 |
| **密钥登录** | 高 | 私钥签名验证，推荐使用 |

### 本地 ~/.ssh/ 目录文件说明

| 文件 | 作用 |
|------|------|
| `config` | SSH 客户端配置文件，定义主机别名、私钥路径、用户名等快捷设置 |
| `id_rsa` / `id_ed25519` | **私钥**，用于认证身份，**绝不能泄漏**，权限必须 600 |
| `id_rsa.pub` / `id_ed25519.pub` | **公钥**，放入服务器的 `~/.ssh/authorized_keys` 中 |
| `known_hosts` | 记录连接过的服务器指纹，防中间人攻击 |
| `known_hosts.old` | `known_hosts` 的备份，SSH 自动生成 |

### 密钥登录流程

1. 客户端持有私钥，服务器 `~/.ssh/authorized_keys` 存有对应公钥
2. 客户端用私钥签名 → 服务端在 `authorized_keys` 中逐行匹配对应公钥
3. 匹配成功后用该行公钥验证签名，验证通过则登录成功

### 自己生成密钥对

```bash
# 生成 Ed25519 密钥（推荐）
ssh-keygen -t ed25519 -C "your_email@example.com"

# 将公钥上传到服务器
ssh-copy-id -i ~/.ssh/id_ed25519.pub root@服务器IP

# 登录
ssh root@服务器IP
```

### 使用云厂商提供的 .pem 密钥

云厂商（阿里云、AWS 等）创建实例时生成 `.pem` 密钥文件：

1. 设置正确权限（必须）

```bash
chmod 600 kxh.pem
```

2. 使用密钥登录

```bash
ssh -i kxh.pem 用户名@服务器IP
```

3. 推荐移动到 ~/.ssh/ 目录

```bash
mkdir -p ~/.ssh
mv kxh.pem ~/.ssh/
chmod 600 ~/.ssh/kxh.pem
```

### 自己生成 vs .pem 的区别

| | 自己生成 | .pem 文件 |
|------|---------|------------|
| 私钥来源 | 本地 `ssh-keygen` 生成 | 云厂商生成后下载 |
| 公钥上传 | 手动 `ssh-copy-id` 推送到服务器 | 云厂商创建实例时自动写入 |
| 私钥路径 | `~/.ssh/id_ed25519`（默认） | 任意位置，每次需 `-i` 指定 |
| 算法 | 通常 Ed25519（更快更安全） | 通常 RSA |

本质都是 SSH 密钥登录，只是密钥对的生成方不同。

### 多私钥/多公钥并存

**私钥**：不能合并，每个私钥文件只存一个私钥，存放在 `~/.ssh/` 下各占一个文件。

**公钥**：服务器端的 `~/.ssh/authorized_keys` 可以将多个公钥放在同一个文件，一行一个，算法不同互不影响：

```
ssh-ed25519 AAAAC3Nza... 本地生成的公钥
ssh-rsa AAAAB3Nza... 云厂商给的密钥对应公钥
ecdsa-sha2-nistp256 AAAA... 另一把公钥
```

### SSH Config 快捷别名

编辑 `~/.ssh/config`，为不同服务器指定不同私钥：

```
Host aliyun
    HostName 服务器IP
    User root
    IdentityFile ~/.ssh/kxh.pem

Host other
    HostName 另一台IP
    User root
    IdentityFile ~/.ssh/id_rsa
```

之后直接 `ssh aliyun` / `ssh other` 即可。

### 登录指定用户

```bash
ssh 用户名@服务器IP
ssh ubuntu@192.168.1.100
```

不写用户名则默认使用当前本地用户名。

### 各云厂商默认用户名

| 云厂商/系统 | 默认用户名 |
|-------------|-----------|
| 阿里云 | `root` |
| AWS EC2 (Amazon Linux) | `ec2-user` |
| Ubuntu 系统 | `ubuntu` |
| Debian 系统 | `admin` |
| CentOS | `root` |

### 安全加固（可选）

编辑服务器 `/etc/ssh/sshd_config`：

```ini
PasswordAuthentication no         # 禁止密码登录
PermitRootLogin prohibit-password # 禁止 root 密码登录（保留密钥登录）
```

重启 sshd：`systemctl restart sshd`

### Debian 非 root 部署服务

```bash
# 创建专用低权限用户（需一次 sudo）
sudo useradd -m -s /bin/bash agent

# 切换到该用户
sudo -u agent -i

# 在该用户下部署和运行服务
```

保持后台运行的方式：

| 方式 | 优点 | 缺点 |
|------|------|------|
| `systemd --user` | 开机自启、崩溃重启、日志管理 | 需写 service 文件 |
| `tmux` / `screen` | 零配置，随时 attach 查看 | 重启后手动起 |
| `nohup &` | 最简单 | 重启后手动起 |
| `crontab @reboot` | 开机自启 | 进程管理弱 |

### 常见问题

**Git clone 报 HTTP2 framing layer 错误：**

```bash
# 让 Git 改用 HTTP/1.1
git config --global http.version HTTP/1.1
```

**pip / uv 配置阿里云镜像源：**

```bash
# pip
mkdir -p ~/.pip
cat > ~/.pip/pip.conf << 'EOF'
[global]
index-url = https://mirrors.aliyun.com/pypi/simple/
trusted-host = mirrors.aliyun.com
EOF

# uv
echo 'export UV_DEFAULT_INDEX=https://mirrors.aliyun.com/pypi/simple/' >> ~/.bashrc
```
