description: "定义代码任务的工作流：理解 → 对齐 → 执行 → 验证 → 审查。用于所有代码相关任务，包括代码修改、调试、重构、测试、构建、脚本、配置、工程化变更和代码审查等；执行代码任务时总是使用该 skill。"

理解 => 对齐 => 执行 => 验证 => 审查

1. 先规划后执行(理解)
   - 上下文最小搜索，理清现状;
   - 多路径头脑风暴，存疑即确认，不假设;
   - 定义目标，规划极简方案，拆分为原子正交任务，优先并行执行;
2. 意图对齐(对齐)
   - 变更直接对应用户需求，禁止范围外修改;
   - 遵循现有代码模式，复用优先;
3. 简洁优先(执行)
   - 改动聚焦最小问题，使用最少代码完成;
   - 避免不必要抽象，控制复杂度;
   - 关键路径使用最少代码注释;
4. 循环验证(验证)
   - 任务独立可测，完成即验证，失败即修正;
   - 同一修正3次失败最小化回滚，记录原因/影响范围，与用户对齐;
5. 影响分析(审查)
   - 发现并修复潜在问题;
   - 分析改动影响范围, 检查连锁影响;
   - 回归核心路径, 确保未引入新问题;
   - 追溯根因, 治本非治标;
