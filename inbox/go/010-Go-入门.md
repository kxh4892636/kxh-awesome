# Go 入门

## 语言定位

### 核心特性

#### 概念

- Go: 编译型、静态类型、自动垃圾回收的通用编程语言;
- 编译型: 源码先编译为机器可执行文件, 再由操作系统运行;
- 静态类型: 变量、表达式、函数参数、返回值在编译期具有确定类型;
- 自动垃圾回收: runtime 回收不再使用的堆内存;
- 并发原语: goroutine、channel、select 由语言和标准库直接支持;
- 标准库: 覆盖 IO、网络、编码、加密、测试、并发等基础能力;

### 程序形态

#### 分类

| 形态       | 核心条件                       | 用途               |
| ---------- | ------------------------------ | ------------------ |
| 可执行程序 | `package main` + `func main()` | 生成可运行二进制   |
| 库包       | 非 `main` 包                   | 被其他包导入复用   |
| 单文件程序 | 一个 `.go` 文件                | 学习语法、验证想法 |
| 多包项目   | 多目录多 package               | 组织真实项目代码   |

## 开发环境

### 安装方法

#### 概念

- 官方安装入口: <https://go.dev/doc/install>;
- 官方安装包: go.dev 提供 macOS package、Windows MSI、Linux archive;
- PATH: shell 查找 `go` 命令的目录列表;
- `/usr/local/go`: 官方安装包默认安装目录;
- Homebrew: macOS 常用包管理器, 非 Go 官方安装方式;

#### macOS

```bash
brew install go # 使用 Homebrew 安装 Go
go version # 验证 go 命令可用
```

```text
go.dev 下载 macOS package -> 打开 .pkg 文件 -> 按安装器提示完成安装 -> 重启 Terminal -> 执行 go version
```

#### Linux

```bash
sudo rm -rf /usr/local/go # 删除旧版官方安装目录
sudo tar -C /usr/local -xzf go<version>.linux-amd64.tar.gz # 解压新版 Go 到 /usr/local/go
echo 'export PATH=$PATH:/usr/local/go/bin' >> ~/.profile # 将 go 命令加入 PATH
source ~/.profile # 让当前 shell 立即读取 PATH 配置
go version # 验证 go 命令可用
```

#### Windows

```text
go.dev 下载 Windows MSI -> 打开 .msi 文件 -> 按安装器提示完成安装 -> 重新打开 cmd 或 PowerShell -> 执行 go version
```

### 安装验证

#### 命令

```bash
go version # 输出 Go 版本、操作系统、CPU 架构
```

### 环境变量

#### 查看命令

```bash
go env # 输出 Go 工具链当前环境变量
go help environment # 查看环境变量文档
```

#### 常用变量

| 变量        | 描述                             |
| ----------- | -------------------------------- |
| GOOS        | 目标操作系统                     |
| GOARCH      | 目标 CPU 架构                    |
| GOROOT      | Go 工具链安装目录                |
| GOPATH      | 传统工作区路径和部分默认缓存路径 |
| GOPROXY     | module 下载代理                  |
| GOMODCACHE  | module 下载缓存目录              |
| GOCACHE     | 编译缓存目录                     |
| GO111MODULE | module 模式开关                  |

### 交叉编译

#### 命令

```bash
GOOS=linux GOARCH=amd64 go build -o hello-linux main.go # 在当前机器生成 Linux amd64 可执行文件
```

## 第一个程序

### 程序结构

#### 语法格式

```go
// 当前文件属于 main 包; main 包可编译为可执行程序
package main

// 导入标准库 fmt 包; fmt 提供格式化输入输出能力
import "fmt"

// main 函数为可执行程序入口
func main() {
	fmt.Println("Hello, Go") // 调用 fmt 包导出的 Println 函数并输出文本
}
```

### 运行源码

#### 命令

```bash
go run main.go # 临时编译并立即运行 main.go
```

### 编译产物

#### 概念

- 可执行文件: 运行时不要求目标机器安装 Go 工具链;
- `go run`: 适合学习、调试、小示例;
- `go build`: 适合生成可分发产物;

#### 命令

```bash
go build -o hello main.go # 编译 main.go 并输出名为 hello 的可执行文件
./hello # 运行当前目录下的 hello
```

## 源码规则

### 注释

#### 语法格式

```go
// line comment; 注释到当前行结束

/*
block comment; 注释可跨越多行
*/
```

### 标识符

#### 概念

- identifier: 程序实体名称, 如变量名、函数名、类型名、包名;
- 命名字符: Unicode 字母、Unicode 数字、下划线;
- 首字符: 必须是 Unicode 字母或下划线;
- 大小写敏感: `name` 和 `Name` 是不同标识符;
- blank identifier: `_`, 接收但丢弃一个值;

#### 语法格式

```go
var userName = "tom" // 普通标识符
_ = userName         // _ 丢弃不需要的值
```

### 关键字

#### 概念

- keyword: Go 语法保留词, 不能作为普通标识符;
- predeclared identifier: 预定义标识符, 可被遮蔽但不推荐;

#### 关键字表

| 分类 | 关键字 |
| ---- | ------ |
| 声明 | `const`, `func`, `import`, `package`, `type`, `var` |
| 控制流 | `break`, `case`, `continue`, `default`, `defer`, `else`, `fallthrough`, `for`, `go`, `goto`, `if`, `range`, `return`, `select`, `switch` |
| 类型/并发 | `chan`, `interface`, `map`, `struct` |

### 文件规则

#### 概念

- Go 源文件: 文件名通常以 `.go` 结尾;
- package 声明: 每个 `.go` 文件第一条非注释语句通常为 `package`;
- 同一目录: 除测试包外通常只声明一个 package;

### 换行规则

#### 概念

- 分号插入: 编译器根据换行位置自动插入分号;
- 花括号位置: 控制语句和函数声明的左花括号通常放在同一行;

#### 语法格式

```go
// 左花括号与函数声明同一行; Go 编译器会按规则自动插入分号
func main() {
	fmt.Println("ok")
}
```
