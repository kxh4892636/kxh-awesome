# 通用飞书文档质量规则

读取、整理、创建、润色、补充或更新任意飞书文档时使用本规则。它定义内容质量、富文本选择、引用文档处理、内部跳转和更新安全要求。

## 全局写作原则

### 读者与主线

- 读者优先：文档结构、语言和素材选择都要降低理解成本，帮助读者快速理解和行动。
- 标题下先定调：重要章节标题下可使用高亮块或首段概述说明结论、背景、范围和阅读路径。这部分可以适当展开，帮助读者建立整体认知。
- 结论前置：章节、表格或关键段落先说明结论、定位或作用，再补充来源、细节和验证方式。
- 细节分层：正文保留主线信息；复杂背景、补充说明、验证方式和参考资料放到表格、列表、引用或附录中。

### 语言与表达

- 正文保持精简：概述之后尽量短句化、分点化，在语义不失真的前提下使用最少文字。
- 分点表达：列表项优先采用“总结 + 简短概述”格式，先给关键词或判断，再用一句话补充说明。
- 单点聚焦：每个章节只回答一个核心问题，每个列表项只表达一个判断或动作。
- 删除噪音：保留必要上下文，删除重复、口语化寒暄、未闭环讨论和无用截图。

### 结构与富文本

- 结构承载逻辑：对比、清单、步骤用表格；流程、链路、关系用画板；截图和 UI 用图文结合。说明文字只补关键判断。
- 富文本优先：优先使用飞书富文本能力承载信息，避免把图片、画板、表格、代码块、看板和引用降级成纯文本。
- 图片使用原图：复用或迁移文档图片时，始终从源文档下载或复制原始图片资源，不用对截图再次截图、飞书预览图、画板导出图、压缩 JPG、缩略图或二次裁剪图替代。若图片要写入画板，先把原图作为目标画板的 media 上传，再使用该白板 media token 创建图片节点。
- 表格自解释：表格不只罗列字段、链接或截图，应补充定位、用途、影响或承接关系，让每行都能独立理解。
- 概念前置：后文依赖专业名词、业务口径、系统模块或数据指标时，先给出简短解释。
- 图文一致：正文、表格、画板、截图说明使用同一套分层、命名和阅读顺序，避免读者在不同表达之间重新映射。
- 导航优先：图片、看板、表格如果过多，先用一张总览图或索引表帮助读者导航。

### 来源与严谨

- 区分事实、推断、建议和待确认项。来源文档明确写出的内容按事实表达；根据上下文整理的内容用“建议”“通常”“优先”“需确认”等限定词。
- 不编造负责人、时间点、权限、平台能力、数据口径、结论或引用来源。
- 事实可追溯：关键判断要能追溯到引用文档、表格、截图、数据看板、会议结论、代码路径或 Trace/logId。
- 引用要转化：引用文档、看板、图片或代码资料要提炼为当前文档可直接使用的结论、规则、入口或素材，并说明用途。
- 涉及数据时写清楚指标定义、时间范围、口径、筛选条件和数据来源。
- 涉及流程时写清楚输入、操作、产出、验收标准和异常处理。
- 涉及风险时写清楚影响面、触发条件、缓解措施和负责人或待定责任方。

### 可行动与可验证

- 读者需要做事时，文档必须给出下一步：谁做、做什么、在哪里做、做到什么程度算完成。
- 行动项用表格或 checkbox，包含事项、负责人、截止时间、依赖、状态。
- 操作手册类文档用“步骤 / 操作 / 结果 / 截图或链接 / 验收点”表达。
- 问题索引用“问题 / 参考”两列；参考列分点列出本文档跳转和引用文档。
- 数据看板要同时给出链接、用途、筛选口径和验证方式。
- 写完后验证标题层级、表格、图片、画板、链接、引用和关键内容是否完整可读。

## 标题和编号

- 飞书文档中不要手写章节序号，例如不要写 `1.3 问题索引` 或 `4 混排组件接入`。
- 使用飞书自动编号：标题 XML 中保留或新增 `seq="auto"` 和 `seq-level="auto"`。
- 新章节标题层级要和上下文一致：
  - 文档核心模块用 `h1`；
  - 模块内主分区用 `h2`；
  - 示例、代码参考、补充说明用 `h3`；
  - 不为了视觉效果跳级。
- 插入新章节前先读取大纲，确认它应放在哪个标题或资源块之后。

## 引用文档处理

读取引用文档时提取：

- 文档标题、doc/wiki token、关键章节，用于生成 cite 或链接。
- 可复用的事实、流程、决策、截图、表格、代码块、画板、看板。
- 平台入口、后台地址、数据看板、Trace、验证插件、配置页面等链接。
- 注意事项、限制条件、验收方式、排障方法和常见误区。

写入目标文档时：

- 引用文档优先用飞书 cite 组件；无法 cite 时用普通链接。
- 不只写“参考某文档”，要说明读者应该看引用文档的哪一部分或用于验证什么。
- 多个引用文档重复时，目标文档保留整合后的最优版本，避免复制粘贴式堆叠。
- 如果引用文档中有图片、画板或看板能直接解释问题，优先复用富文本素材；复用图片时必须使用原始图片资源，而不是预览图或二次导出的替代图。

## 内部跳转链接

当用户要求“点击能跳转到对应位置”或文档较长时：

- 先读取目标文档 `outline --detail with-ids`，拿到标题 block id。
- 链接格式优先使用目标文档 URL + `#block_id`。
- 表格中建议写成：
  - `本文档：` `<a href="文档URL#block_id">章节名 / 小节名</a>`
  - `引用文档：` `<cite ...></cite>` 或普通链接
- 跳转到具体提示块、画板或表格时，使用对应 block id；跳转到流程中的某一步但没有独立 block id 时，跳到所属小节。

## 富文本选择规则

富文本用于降低理解成本，不只是装饰。优先选择能承载信息结构的载体，只有当信息足够简单时才使用普通段落。

| 内容类型 | 推荐表达 |
| --- | --- |
| 结论、定位、重要原则 | callout |
| 操作流程、行动项、问题索引 | table |
| 总体链路、架构、状态流、因果关系 | whiteboard 或保留原画板 |
| 多张截图对比 | table 或 grid |
| 决策项、风险项、上线检查 | checkbox 或 table |
| 代码、配置、命令 | pre/code，保留语言类型 |
| 平台入口、看板、外部资料 | 普通链接、书签链接或 cite |
| 来源文档 | cite |

## 通用改写模板

适用于把零散文档改成结构化文档：

```xml
<h1 seq="auto" seq-level="auto">总览</h1>
<callout emoji="💡">
  <p><b>结论：</b>用 1-3 句话说明本文最重要的信息。</p>
</callout>
<h1 seq="auto" seq-level="auto">背景与目标</h1>
<h1 seq="auto" seq-level="auto">核心内容</h1>
<h1 seq="auto" seq-level="auto">流程或方案</h1>
<h1 seq="auto" seq-level="auto">风险与待确认</h1>
<h1 seq="auto" seq-level="auto">参考资料</h1>
```

## 问题索引模板

问题索引用于帮助读者快速决定看哪一节。保持两列，但参考列要分点展示。

```xml
<table>
  <thead>
    <tr><th>问题</th><th>参考</th></tr>
  </thead>
  <tbody>
    <tr>
      <td>
        <p><b>问题短标题</b></p>
        <p>使用者常见困惑，用一句话描述。</p>
      </td>
      <td>
        <ul>
          <li><b>本文档：</b><a href="目标文档URL#block_id">章节 / 小节</a></li>
          <li><b>引用文档：</b><cite type="doc" ...></cite></li>
        </ul>
      </td>
    </tr>
  </tbody>
</table>
```

## 更新飞书文档时的安全规则

- 默认做局部更新，不使用 `overwrite`，除非用户明确要求重建全文。
- 更新前读取最新 revision 和目标 block，防止覆盖用户刚改的内容。
- 不删除未读懂的图片、表格、画板、引用、评论上下文。
- 优化已有看板或表格时，优先在原结构上补字段、补说明、补链接，不随意重排全部内容。
- 写入 XML 时只转义文本里的 `<`、`>`、`&`，不要转义标签本身。
- 表格、列表、callout、pre/code 写入后必须重新读取目标章节确认飞书解析正常。
